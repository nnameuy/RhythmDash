# 项目设计文档：RhythmDash

---

本项目是一款节奏跑酷游戏，有一定的灵感来源于Geometry Dash，不过实际内容有较大区别。其核心特性在于**无需人工设计关卡**，通过音频处理算法以预处理 MP3 文件，从音频中提取音乐节奏和不同特征的节奏点，并根据此自动生成与之匹配的游戏障碍物。

#### 1. 我们期望达成的目标

我们希望运行游戏主体之后，进入一个游戏的初始菜单。

| 选项       | 说明                             |
| -------- | ------------------------------ |
| Select   | 在已经生成好的关卡里选择一张游玩               |
| Generate | 上传.mp3音频，并生成一个关卡。点击后会出现五个难度选项。 |
| Settings | 按键自定义、音量调节等                    |

首先是**音频提取部分**（生成地图）。

> 需要从输入的原始MP3音频中提取特征（一般是一段有节奏感的音乐），主要是听感明显的特征，例如鼓点。
> 
> 难度分为**BASIC**、**ADVANCED**、**EXPERT**、**MASTER**、**Re:MASTER**五个难度，程序为每段音频生成五种难度的地图，难度选择的不同生成密度依次增大的时间戳。
> 
> 这部分的重大挑战是**控制延迟与误差**，因为将来我们希望障碍物/敌对实体出现在相应物理位置的时机，与对应音频节奏的偏差足够低。这个游戏希望设计为亚音游，不过按照一般音乐游戏的经验，这个误差大概需要控制到5ms以下才能提供较正常的游玩体验。

然后是根据上一步提取出的特征**自动设计关卡**。

> 我们的设计的核心逻辑是，对于向左运动速度恒定的障碍物/敌对实体，玩家需要做出一定的反应：例如躲避或攻击等等。
> 
> 我们希望通过关卡设计来引导玩家在**某个最佳的时机**进行上述反应，这个最佳时机是与最开始提取出的音频特征点同步的。即我们希望玩家随着音乐节奏做出对障碍物/敌对实体的反应操作。

随后是**游戏主体部分**。

- Part.A **玩家操作**

> 首先我们能以两个按键控制玩家角色的运动方向：左上和右下。在操控下玩家以折线轨迹在地图上前进。玩家通过改变轨迹来进行躲避操作。
> 
> 然后还会有一个按键用于击打敌对实体，这里可以进行一些多样化设计，待定。
> 
> 而后是评分规则，我们以完成度为评分标准。

- Part.B **角色与场景**

> 如果玩家与障碍物/敌对实体发生碰撞，则终止游戏。
> 
> 玩家的击打可以使敌对实体失效。

- Part.C **画面特效**（待补充）

> 根据节奏的强弱加入适当的闪光/色彩高速变化等效果。
> 
> 在强节奏点加入震动效果（画面颤动）
> 
> 发光粒子效果。

#### 2. 不同模块的实现

*该部分计划使用Python完成*

- **音频处理部分**：仅有这部分计划使用python语言实现。
  
  利用 `librosa` 、`SciPy.signal`、`NumPy`这三个库联合进行特征提取。
  
  我们计划编写一个独立的 Python 脚本 `analyzer.py`，使用 `librosa` 库读取 MP3。
  
  Python脚本将会提取出节奏点并生成对应的时间戳，我们计划用`.json`文件来存储，py脚本运行结果保存为 `data.json` 。
  
  这里计划使用`NumPy`的快速傅里叶分解（FFT）生成频谱：
  
  - 方便程序识别不同的乐器，不同频段的采音生成不同的物体；相比纯粹的根据音高生成物体，这一方案会好很多。
  
  - 对连续的重低音可以补充震动特效等效果。

*以下部分计划使用C++ with Qt完成*。

- **关卡生成模块**：将时间戳序列转换为游戏世界的实体坐标。

- **游戏主体部分/引擎**：处理玩家物理状态和位置，以及进行碰撞检测。

- **可视化**：基于 `Qt` ，实现游戏的可视化。
