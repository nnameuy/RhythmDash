# RhythmDash 设计规范

## 1. 难度设计

### 1.1 五档 intensity 阈值

所有音频事件提取完成后，按 intensity 值过滤以生成不同难度：

| 难度 | intensity 阈值 | 密度描述 |
|------|---------------|----------|
| BASIC | intensity ≥ 0.70 | 仅最强节奏点 |
| ADVANCED | intensity ≥ 0.50 | 加入中等事件 |
| EXPERT | intensity ≥ 0.35 | 加入大部分事件 |
| MASTER | intensity ≥ 0.20 | 近乎全部，过滤噪音 |
| Re:MASTER | 保留全部 | 最大密度 |

### 1.2 设计原则
- **intensity 阈值只过滤 entity_stream**，macro_stream 在所有难度中保持一致（氛围效果与难度无关）
- 相同 `time_ms` 可以有多个不同事件（例如同一时刻同时有 kick 和 snare）

## 2. 实体类型 → 游戏对象映射（草案）

| 音频事件类型 | 游戏障碍物 | 玩家反应 |
|-------------|-----------|----------|
| onset_kick | 地面障碍物 | 跳跃躲避 |
| onset_snare | 空中障碍物 | 下蹲躲避 |
| onset_hihat | 小型飞行物 | 微调位置 |
| onset_crash | 大型障碍物 | 大幅移动 |
| hold_bass | 持续地面障碍 | 保持跳跃 |
| hold_melody | 持续空中障碍 | 保持下蹲 |

> 注：此映射为初版设计，后续可能根据原型测试调整。

---

## 3. 宏观流 → 画面特效映射（草案）

| 宏观事件类型 | 画面特效 |
|-------------|----------|
| energy_buildup | 屏幕逐渐变亮 / 粒子加速 |
| energy_drop | 闪光 + 画面震动 |
| silence_gap | 画面变暗 / 时间减速感 |

---

## 4. 玩家操作规范

| 按键 | 功能 |
|------|------|
| 方向键 ↑ / W | 向右上移动 |
| 方向键 ↓ / S | 向右下移动 |
| 空格 / J | 攻击敌对实体 |

- 玩家移动轨迹为折线，在两条水平线之间切换
- 默认情况下角色自动向前（向右）移动

---

## 5. 视觉风格（原型阶段）

- 极简几何风格：纯色块 + 几何图形
- 玩家：一个可见的几何形状（如方块或三角形）
- 障碍物：不同颜色的几何形状（按类型区分颜色）
- 背景：深色纯色或简单渐变
- 后续可替换为美术资源
